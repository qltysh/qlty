# Clippy

[Clippy](https://github.com/rust-lang/rust-clippy) is the official Rust linter — a collection of lints to catch common mistakes and improve Rust code quality.

## Enabling Clippy

Enabling with the `qlty` CLI:

```bash
qlty plugins enable clippy
```

Or by editing `qlty.toml`:

```toml
# Always use the latest version
[[plugin]]
name = "clippy"

# OR pin to a specific version
[[plugin]]
name = "clippy"
version = "X.Y.Z"
```

## Auto-enabling

Clippy will be automatically enabled by `qlty init` when Rust files are present.

## Languages and file types

Clippy analyzes: Rust (`.rs`). It requires a `Cargo.toml` in the scanned directory or a parent directory.

## Configuration files

- [`clippy.toml` / `.clippy.toml`](https://doc.rust-lang.org/clippy/configuration.html)

To keep your project tidy, you can move configuration files into `.qlty/configs` and Qlty will find and use them when running Clippy.

## Troubleshooting

**"failed to parse lock file" error.**
Cargo cannot read `Cargo.lock`, usually because it was generated by a newer version of cargo or has become corrupted.
Regenerate it with `cargo generate-lockfile`, or delete it and run `cargo build` to recreate it.
